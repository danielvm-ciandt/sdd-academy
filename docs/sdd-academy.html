01 — How It Works

Pick a method. Pick a project. Build it.

Three steps to learn spec-driven development by doing. No lectures, no theory-only sessions. You learn by building real systems against real databases using the method of your choice.

Browse the 9 SDD methods below, or use the Method Selector to find the best fit for your project type, team size, and experience level. Each method has a complete "From Zero to Prod" guide.

03

AI-Assisted IDEs & Tools Reference

Cursor

AI-first code editor built on VS Code. Native Composer for multi-file edits, .cursor/rules for project context, MCP support. Best for visual developers who want IDE integration.

cursor.com

Claude Code

Anthropic's CLI agent for agentic coding. Terminal-first, supports slash commands, MCP servers, CLAUDE.md project context. Best for developers who prefer terminal workflows.

docs.anthropic.com

GitHub Copilot

GitHub's AI pair programmer. Works in VS Code, JetBrains, Neovim. Inline completions and chat. Wide language support. Best for teams already in the GitHub ecosystem.

github.com/features/copilot

Windsurf

Codeium's AI IDE (formerly Codeium Editor). Cascade for multi-file flows, MCP support, agent mode. Good balance of visual IDE and agentic capabilities.

codeium.com/windsurf

Cline

Open-source autonomous coding agent for VS Code. Supports Claude, GPT, Gemini, and local models. MCP support, file creation, terminal commands. Best for open-source advocates.

github.com/cline/cline

Antigravity

AI coding platform focused on large-scale codebases. Context engine for repository understanding. Ideal for enterprise teams working with monorepos and complex architectures.

antigravity.dev

Aider

Open-source CLI pair programming tool. Git-aware, supports many LLMs, conversational editing. Lightweight and scriptable. Best for developers who want minimal tooling overhead.

aider.chat

Continue

Open-source AI code assistant for VS Code and JetBrains. Custom model support, context providers, slash commands. Best for teams wanting full control over their AI stack.

continue.dev
06 — Team Leader Playbook

Rollout Strategies for Team Leaders

Six proven strategies for team leaders to drive SDD adoption across their squads. Mix and match based on your team's culture and learning style.

⚔

Method Deathmatch

Two team members pick different methods, same Neon database, same time limit. They compare results using the scoring rubric. Creates engagement and real comparative data.

  • Pair two developers with different experience levels
  • Both pick the same Neon database project
  • Each picks a different SDD method
  • Timebox to 4 hours
  • Present results: working code, specs quality, and lessons learned
  • Team votes on "most productive method" for that project type
📚

Method of the Month

Focus on one SDD method per month. That month's method gets a Slack channel, a weekly office-hours call, and a challenge. By month 9, you have organic champions.

  • Month 1: Start with OpenSpec or SpecPulse (easiest onramp)
  • Create #sdd-[method-name] Slack channel
  • Weekly 30-min office hours with early adopters
  • End-of-month showcase: 3 best implementations presented to team
  • Rotate to next method and carry forward champions
🎓

Teach-Back Sessions

After someone completes a method+project combo, they present a 15-minute teach-back to their squad. Teaching is the fastest path to mastery.

  • Developer completes a method + project combination
  • Prepares 15-min presentation: what worked, what didn't, key learnings
  • Shows their repo: spec artifacts, code structure, test coverage
  • Team asks questions and discusses applicability to current projects
  • Add insights to the team's internal knowledge base
🏆

SDD Leaderboard

Opt-in leaderboard showing completions per team member. Three completed method+project combos = "SDD Practitioner". Gamify the learning path.

  • Define tiers: Explorer (1 combo), Practitioner (3), Expert (6), Master (9)
  • Track completions with GitHub repo submissions
  • Monthly recognition in team all-hands
  • SDD Masters become mentors for newcomers
  • Aggregate data to identify which methods work best for your team
🚀

Challenge of the Week

Weekly micro-challenge: a specific method + database + constraint (e.g., "Build a REST API for Chinook using TDD in AI-SDD, timebox 3 hours"). Creates social momentum.

  • Post challenge every Monday in #sdd-academy channel
  • Include: method, database, constraint, timebox
  • Developers share repos by Friday
  • Monday review: quick walkthrough of 2-3 submissions
  • Best solution gets "SDD Pick of the Week"
🛠

Real Project Pilot

After the team tries 2-3 methods on practice projects, pick one method and apply it to a real client sprint. Nothing validates learning like production use.

  • Select a low-risk feature in a real client project
  • Team votes on which method to pilot
  • Assign an SDD Expert as the "method coach" for the sprint
  • Run a retro: compare spec quality, build time, defect rate vs. previous sprints
  • Document findings as an internal case study
  • Share results at team all-hands for broader adoption
